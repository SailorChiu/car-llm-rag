# Agent / 工具调用 Demo

> **一句话总结**：本 demo 演示了 意图识别 → 工具调用（RAG / mock 车控）→ 结果回填 → 多轮记忆 的闭环，
> 用 Qwen2.5-1.5B 实现 function-calling 风格的 Agent 循环。

---

## ⚠️ 免责声明

**车控函数（`set_climate` / `control_window`）均为 MOCK / 演示用，非真实车控。**
打印日志并返回假状态，不向任何车辆系统发送指令。
真实场景需对接 CAN 总线 / OEM 车控 API（如 AUTOSAR Adaptive Platform 标准接口）。

---

## 🚀 快速启动

```bash
cd ~/car-llm && source .venv/bin/activate

# 确保 RAG 索引已构建（只需跑一次，已构建可跳过）
bash run.sh rag-build

# 启动 Agent
python agent/agent_demo.py
```

首次加载 LLM 约需 30–60 秒（与 `rag-chat` 相同）。输入 `q` 或 Ctrl-C 退出。

## 🏗️ 架构图（文字版）

```
┌─────────────────────────────────────────────────────────┐
│                   用户输入（自然语言）                    │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│             Step 1：意图识别（LLM 第一次调用）            │
│                                                         │
│  System Prompt（工具列表 + JSON 格式要求 + 记忆 hint）   │
│  + 多轮历史 messages                                     │
│  → Qwen2.5-1.5B greedy decoding                         │
│  → 输出 JSON：{"tool": "...", "args": {...}}             │
│                                                         │
│  ⚙️  容错：正则抠 JSON → parse 失败 → fallback chitchat  │
└────────────────┬────────────────────────────────────────┘
                 │
        ┌────────┴──────────┐
        │    工具路由(dispatch) │
        └────────┬──────────┘
                 │
    ┌────────────┼────────────────────┐
    ▼            ▼                   ▼
query_manual  set_climate        control_window
(RAG 检索)   ⚠️MOCK / 演示      ⚠️MOCK / 演示
    │            │                   │
bge-small-zh  MEMORY dict        MEMORY dict
+ FAISS       记录温度              记录操作
+ 共享LLM生成
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│           Step 2：结果回填（LLM 第二次调用）              │
│                                                         │
│  工具执行结果 → Observation 注入 messages               │
│  → Qwen2.5-1.5B 生成最终自然语言回复                    │
│  （query_manual 内部已完成此步，直接返回）                │
└────────────────┬────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────┐
│              最终回复 → 打印给用户                        │
│  history.append(user_msg + final_reply)   ← 多轮持续    │
│  MEMORY dict 跨轮保存用户偏好              ← 记忆机制    │
└─────────────────────────────────────────────────────────┘
```

**关键工程决策**：

| 决策 | 原因 |
|------|------|
| 共享 LLM 实例 | `query_manual` 复用同一份权重，8G 显存约束下只占一份（~3.5G fp16） |
| 正则抠 JSON + fallback | 1.5B 小模型偶尔夹带多余文字，`re.search` 兜底，parse 失败不崩溃 |
| MEMORY dict 注入 system hint | 多轮对话中显式告知模型"上次温度=22°C"，跨轮引用更可靠 |
| greedy decoding（Step 1）| 工具选择需要确定性输出，do_sample=False |

## 📋 验收示例（真实运行输出）

> 在 WSL2 · RTX 4060 Laptop 8G · Qwen2.5-1.5B-Instruct 上实测，无任何编造。

```
==============================================================
  车载智能助手 Agent Demo
  (工具调用 · 多轮对话 · 记忆机制)
==============================================================
  输入 'q' 或 Ctrl-C 退出

加载 LLM：Qwen/Qwen2.5-1.5B-Instruct …（首次加载约需 30-60s）
✓ 模型已加载（device=cuda:0）

你：前挡风起雾了怎么处理
  ▶ 意图识别 → [query_manual]  args={"question": "前挡风起雾了怎么处理"}
  ▶ RAG 命中知识块：['空调系统', '雨刮', '灯光系统']

助手：前挡风玻璃起雾时，按下前除雾键，空调会自动切到除雾模式并打开压缩机。

──────────────────────────────────────────────────────────────
你：把温度调到22度
  ▶ 意图识别 → [set_climate]  args={"temperature": 22}
[MOCK-车控] set_climate → 22.0°C  （演示用，非真实车控，不接入任何 CAN/OEM 接口）

助手：现在车内温度已经调整到了22摄氏度。

──────────────────────────────────────────────────────────────
你：刚才我设的多少度来着
  [解析] JSON 提取失败，原始输出：'您之前设定的温度是22.0°C。'
  [解析] fallback → chitchat
  ▶ 意图识别 → [chitchat]  args={}

助手：您之前设定的温度是22.0°C。

──────────────────────────────────────────────────────────────
你：q
再见！
```

### 结果解读

| 轮次 | 输入 | 路由 | 验证点 |
|------|------|------|--------|
| 1 | 前挡风起雾了怎么处理 | `query_manual` → RAG | 命中「空调系统/雨刮」知识块，回答准确 |
| 2 | 把温度调到22度 | `set_climate(22)` → MOCK | 温度记入 MEMORY，MOCK 标注清晰 |
| 3 | 刚才我设的多少度来着 | `chitchat`（fallback） | **模型直接从记忆/上下文读出 22°C**，容错机制触发但结果正确 ✅ |

> 第三轮的 fallback 本身就是亮点：模型跳过了 JSON 格式直接回答，
> 说明记忆注入（system hint）让小模型不需要走工具路由也能正确引用——
> 兜底逻辑使 demo 在小模型格式不稳时仍能正常工作。
